# Phase 5 Post-MVP Expansion

Status: prototype complete as of 2026-05-25.

## Implemented Scope

- Dedicated `/expansion` workspace for the post-MVP roadmap.
- Readiness gate for starting Phase 5: 5-10 paying merchants actively using hosted pages.
- Expansion readiness signals for:
  - Paying hosted-page merchants.
  - Active public hosted pages.
  - First-month renewal proof.
  - Repeated integration requests.
- Ordered feature roadmap matching the product plan:
  1. Simple embed snippet or iframe.
  2. Shopify app.
  3. WooCommerce plugin.
  4. Product variant/material configurator.
  5. Merchant analytics dashboard v2.
  6. Bulk product import.
  7. White-label pages.
  8. API access.
  9. Native SDK.
  10. Construction/project showcase product.
- Each expansion feature includes merchant value, trigger, MVP scope, status, and blockers.
- Construction remains explicitly deferred into a separate vertical and starts from uploaded existing 3D/CAD files, not photo-to-building generation.

## Product Rules Preserved

- Hosted public product pages remain the core proven product.
- Integrations are sequenced by merchant demand, not treated as hidden MVP requirements.
- The first expansion step is the lowest-lift embed/iframe surface.
- Shopify and WooCommerce follow only after hosted pages and embeds show repeat paid demand.
- API, SDK, and construction remain deferred until e-commerce revenue and platform contracts are clearer.

## Deferred

- Production iframe rendering and domain allowlisting.
- Shopify OAuth, app listing, billing, and theme app block implementation.
- WooCommerce packaging, distribution, and WordPress admin UI.
- Production analytics ingestion, charting, and export.
- Bulk CSV import execution and batch generation queue controls.
- White-label subdomain provisioning and DNS support.
- Public API authentication, rate limiting, documentation, and SDKs.
- Construction-specific validation, CAD conversion, and project-page workflows.
